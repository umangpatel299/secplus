MSGMessage-Based Attack OverviewMost Common Vector

Message-based attacks exploit the communication channels people use daily — email, SMS, instant messaging, and collaboration tools. Since messages arrive in a trusted inbox and users must interact with them to do their jobs, they are the most effective initial access vector.

Why Messages Are So Effective

# Why messages work where technical attacks fail: Firewall: blocks network attacks ← cannot block email AV: detects malware ← clean link bypasses AV EDR: detects exploitation ← clicking a link = user action Patch mgmt: fixes known CVEs ← user action not a CVE Email arrives: looks legitimate → user clicks → credential stolen Security tools: saw nothing wrong → no alerts → no blocks # Message attack kills: trust + urgency + familiarity Trust: "From your bank" / "From IT department" / "From your boss" Urgency: "Your account will be closed" / "Respond immediately" Familiar: looks exactly like a real email from the real service # #1 initial access vector for APTs, ransomware, BEC, all major attack types # Defending against it = combination of technical + human controls

// Exam Keywords

Message-based vector →#1 initial access method — email, SMS, IM — exploits human trust
Why effective →Bypasses technical controls — relies on human action — trust + urgency + familiarity
MSGEmail Attack TypesPhishing · Spear · Whaling · BEC
🎣
Phishing — Mass Email Campaign
Generic emails sent to millions of recipients. No personalization — designed for volume. "Your PayPal account has been suspended." High quantity, low success rate per email, but the math works at scale. Modern phishing kits: free on dark web — anyone can run a campaign. Contains: malicious link (fake login page), malicious attachment, or both.
🏹
Spear Phishing — Targeted Attack
Personalized email targeting specific individual using OSINT. References real details: manager's name, current project, company events. "Hi Sarah — following up on the Q4 forecast you discussed with David yesterday." Much higher success rate — harder to spot. Primary initial access method for APTs and sophisticated ransomware groups.
🐋
Whaling — Executive Targeting
Spear phishing targeting CEO, CFO, CISO, or other senior executives. High-value target — executive credentials open everything. Tactics: impersonate board member, attorney, regulator. Exec targeted with highly professional email relevant to their role. Also: attackers impersonate executives to target subordinates (BEC variant).
💼
Business Email Compromise (BEC)
Attacker compromises or spoofs executive email to request fraudulent wire transfers or gift cards. "From: CEO" to CFO: "I need you to wire $250K urgently for a confidential acquisition — don't discuss with anyone." FBI: $2.7B losses in 2022. Defense: dual-approval for all transfers + out-of-band verification + no "secrecy" bypass of process.
📦
Malicious Attachments
Office documents with macros (Word, Excel), PDFs with embedded JavaScript, ZIP/RAR files containing executables, LNK files (Windows shortcuts) that execute PowerShell. Modern defenses block .exe — attackers use: .docm, .xlsm, .iso, .lnk, .html smuggling. Email sandbox detonates attachments before delivery.
🔗
Malicious Links — URL Manipulation
Links that look legitimate but lead to attacker-controlled sites. Techniques: typosquatting (paypa1.com), homograph attacks (рaypal.com with Cyrillic р), URL shorteners (hide real destination), open redirects (legitimate site redirects to malicious one), subdomain tricks (paypal.attacker.com). Always hover before clicking — check the actual URL in status bar.

SLAM — Detection Method

S — Sender

Check full email address — not just display name. Display name can be anything. Domain: microsofft vs microsoft. Hover to see real sending domain.

L — Links

Hover before clicking — see real destination in status bar. Does URL match claimed site? Is it a URL shortener hiding the destination?

A — Attachments

Expecting this file? Is file type suspicious (.docm, .xlsm, .iso, .lnk)? When in doubt: don't open — report to IT.

M — Message

Urgency without time to think? Threatening consequences? Request for unusual action (wire transfer, gift cards)? Secrecy request?

MSGSMS, IM & Collaboration Platform AttacksSmishing · Teams · Slack

Smishing — SMS Phishing

Phishing via text message. "USPS: Your package requires confirmation — click here." Mobile users click links more readily — harder to verify URLs on small screen. Phone number spoofing makes sender appear legitimate. Rising vector — MFA bypass: "Your verification code is... text STOP to receive no more texts" — trick users into providing codes. Defense: don't click unexpected SMS links, verify via official app.

Collaboration Platform Attacks

Microsoft Teams, Slack, Zoom: increasingly targeted. External user sends malicious file or link via Teams. Attacker compromises one account → messages all contacts with malicious link (appears to come from trusted colleague). "Hey [name] — can you review this quickly?" + malicious link. Often lower security scrutiny than email — employees trust IM more. Defense: same scrutiny as email for unexpected links/files.

QR Code Phishing (Quishing)

Malicious QR codes in emails, flyers, or physical stickers placed over legitimate codes. Email scanners check links — cannot scan QR code destination. User scans with phone → lands on phishing site → phone has weaker defenses. Physical: sticker over payment terminal QR code → attacker's page. Defense: scan QR codes with URL preview enabled, verify destination before proceeding.

Email Header Spoofing

Send email with forged From: header appearing to be from legitimate domain. Basic spoofing: email "from" [email protected] when actually from attacker. Technical defenses: SPF (specifies authorized sending servers), DKIM (cryptographically signs email), DMARC (policy for failing SPF/DKIM). Without these: easy to spoof any domain. With all three: spoofing much harder.

Unsecure Networks
Open WiFi · Rogue AP · Bluetooth · On-path Attacks
NETUnsecure Network OverviewWiFi · MITM · Eavesdrop

Unsecure networks allow attackers to intercept, modify, or inject traffic between victims and legitimate services. Any network where traffic travels unencrypted or where authentication of the network itself is weak becomes an attack surface.

Why Unsecure Networks Are Dangerous

# The coffee shop scenario: Employee connects to: "Starbucks_WiFi" (free, open, no password) Attacker on same network: passive sniffing Unencrypted traffic: HTTP sites: attacker reads everything → credentials in plaintext Unencrypted email (SMTP port 25): attacker reads emails DNS queries: attacker sees every domain you visit HTTPS: encrypted → attacker can't read → but sees domains (SNI) # Even "secure" network can be compromised: Rogue AP: attacker creates "Starbucks_WiFi" with better signal victim connects to attacker's network all traffic flows through attacker → MITM even HTTPS: attacker can attempt SSL strip # Defense: always use VPN on untrusted networks # Even better: ZTNA — verify every session regardless of network

// Exam Keywords

Unsecure network risk →Eavesdropping on unencrypted traffic, MITM attacks, rogue access points
Open WiFi →No authentication — any attacker on same network can sniff traffic
NETWireless Attack TypesEvil Twin · Deauth · WPS
👿
Evil Twin / Rogue Access Point
Attacker creates a WiFi access point with the same SSID as a legitimate one (or better signal). Victims connect to the evil twin instead of the real AP. All traffic flows through the attacker. Can be combined with SSL stripping to downgrade HTTPS to HTTP. Detection: network monitoring for duplicate SSIDs, 802.1X with certificate validation prevents unauthorized APs.
🚫
Deauthentication (Deauth) Attack
WiFi 802.11 management frames (including deauth) are not authenticated in WPA2. Attacker sends spoofed deauth frames to disconnect victims from legitimate AP. Purpose: force client to reconnect — capture WPA2 4-way handshake for offline brute force attack. Also: denial of service against wireless networks. Defense: WPA3 (SAE/PMF protects management frames), 802.11w Management Frame Protection.
📡
WPS Attack
Wi-Fi Protected Setup (WPS) PIN feature has a design flaw: PIN verified in two halves = only 10^4 + 10^3 = 11,000 combinations. Reaver tool brute forces WPS PIN in hours → reveals WPA2 passphrase. Defense: disable WPS on all access points — there is no patch for this design flaw.
🔓
Downgrade Attacks
Force connection to use weaker security than available. SSL stripping: downgrade HTTPS to HTTP by intercepting redirect. KRACK (WPA2 Key Reinstallation Attack): replay attack on WPA2 handshake. Protocol downgrade: force TLS 1.0 instead of TLS 1.3. Defense: HSTS (HTTP Strict Transport Security) prevents SSL stripping — browser refuses HTTP. Minimum TLS version policy.

// Exam Keywords

Evil twin →Rogue AP with same SSID — victims connect — all traffic via attacker
Deauth attack →Spoofed management frames disconnect clients — capture handshake for cracking — WPA3/PMF defends
WPS attack →PIN design flaw — brute forceable — disable WPS entirely
HSTS →HTTP Strict Transport Security — browser refuses HTTP — prevents SSL stripping
NETBluetooth, NFC & RF AttacksBluesnarfing · Bluejacking · NFC

Bluesnarfing

Unauthorized access to data on a Bluetooth device — contacts, calendar, messages, files. Exploits Bluetooth vulnerabilities before pairing is required. Range: typically up to 10m (class 2), extended with directional antenna. Defense: turn off Bluetooth when not needed, keep firmware updated, use Bluetooth 5.0+ with improved security.

Bluejacking

Sending unsolicited messages to Bluetooth devices. Less dangerous than bluesnarfing — no data theft, just unwanted messages. Can be used for social engineering (sending urgent messages to nearby devices). More nuisance than serious attack. Defense: don't accept unexpected Bluetooth connections/messages.

NFC Attacks

Near Field Communication — payment terminals, access cards, contactless transactions. Skimming: read NFC card data without victim knowing (range: a few cm, but specialized hardware extends range). Relay attack: capture and replay NFC transaction at distance. Defense: RFID-blocking wallets/sleeves, transaction limits, tokenization of card data.

RF (Radio Frequency) Attacks

Jamming: flood frequency with noise → disrupts communication (GPS jamming, WiFi jamming, cellular jamming). Replay: capture and replay legitimate RF signal (car key fobs, garage doors). Defense: frequency hopping, rolling codes (car fobs — each signal unique), encrypted RF protocols.

// Exam Keywords

Bluesnarfing →Unauthorized Bluetooth data theft — contacts, messages — exploit BT vulnerabilities
Bluejacking →Unsolicited Bluetooth messages — less dangerous — social engineering potential
NFC skimming →Read contactless card data — RFID-blocking wallet defends — payment tokenization

Social Engineering
Manipulate People · Pretexting · Physical · Psychological Principles
SOCSocial Engineering & Psychological PrinciplesHuman Exploitation

Social engineering exploits human psychology rather than technical vulnerabilities. It is often easier to trick a person than to exploit software. Understanding the psychological principles used helps defenders train employees to recognize manipulation.

Six Psychological Principles Attackers Exploit

Authority

"This is IT support calling — I need your credentials urgently." People comply with authority figures. Attackers impersonate: IT, management, CEO, government, police. Defense: verify identity out-of-band — call back to known number.

Urgency / Scarcity

"Act now or your account will be closed." "Limited time." Time pressure prevents careful thinking. Creates panic — victim acts without verifying. Defense: slow down — legitimate requests don't expire in minutes.

Social Proof

"Everyone else has already updated their credentials." "Your colleagues have all verified." People conform to what others do. Fake consensus creates pressure. Defense: be independently skeptical of claimed consensus.

Familiarity / Liking

"Hi, this is Sarah from accounting — we met at the conference." Build rapport before making the request. People help people they like. Defense: verify identity regardless of how friendly they are.

Intimidation / Fear

"You will be audited / arrested / fired if you don't cooperate." Fear of negative consequence overrides judgment. Defense: verify threat legitimacy — real authorities follow formal processes.

Reciprocity

"I just helped you with that issue — now I need you to help me." People feel obligated to return favors. Attacker does something helpful first → victim feels debt. Defense: recognize obligation feelings as manipulation triggers.

SOCSocial Engineering TechniquesPretexting · Baiting · Quid Pro Quo
Pretexting
Fabricating a convincing backstory to manipulate a target. "I'm calling from your bank's fraud department — we need to verify some transactions." The pretext creates legitimacy. Used to: extract information, gain physical access, get credentials reset. Often combines multiple principles (authority + urgency). Defense: call back to official number — don't use number they provide.
Baiting
Enticing victim with something desirable to trigger a security compromise. USB drop attack: leave USB labeled "Q4 Salaries" in company parking lot — curious employee plugs it in → malware. Online baiting: "Free Netflix account" link → credential phishing or malware. Relies on curiosity and greed. Defense: policy + training against unknown USB devices, endpoint controls blocking unauthorized USB.
Quid Pro Quo
Offer something in exchange for information or access. "If you give me your credentials, I'll fix your computer remotely." Fake tech support: "Call us and we'll fix your slow computer" — then ask for remote access. Uses reciprocity principle. Defense: only work with verified IT support — initiate contact yourself via official channels.
Watering Hole Attack
Compromise a website that the target population regularly visits. Target group: security researchers → compromise popular security blog → all visitors get drive-by malware. Harder to defend than targeted phishing — victim visits legitimate (but compromised) site. Defense: web filtering, keep browsers patched, EDR behavioral detection, network monitoring.
Vishing — Voice Phishing
Social engineering via phone call. "IT support calling — we detected malware on your computer." Caller ID spoofing makes number appear legitimate. AI voice cloning: deepfake voice of known person. Rising attack: AI CEO voice clone calls CFO → requests urgent wire transfer. Defense: verbal challenge code, call back to known number, no credentials over phone.

SOCPhysical Social EngineeringTailgating · Shoulder Surfing

Tailgating / Piggybacking

Following an authorized person through a secure door without scanning own badge. Tailgating: person ahead holds door — "thanks" social pressure. Piggybacking: with victim's knowledge — "I forgot my badge." Mantrap/airlock physically prevents — one person at a time. Policy: challenge unknown persons — "If you see something, say something." Cameras at entrances.

Shoulder Surfing

Physically observing someone enter credentials, PIN, or sensitive information. Coffee shop: watching someone type password. ATM: watching PIN entry. Defense: privacy screens, positioning awareness, shield keyboard when entering PIN. Modern risk: recording with smartphone from distance. Biometric login reduces password shoulder surfing risk.

Dumpster Diving

Searching through physical or digital trash for useful information. Physical: printed documents with account numbers, org charts, network diagrams, access codes written on paper. Digital: deleted files not securely wiped. Defense: cross-cut shredding policy, secure document destruction for all sensitive paper, NIST 800-88 media sanitization for digital.

Impersonation

Physically pretending to be someone else — delivery driver, contractor, IT technician, fire inspector. Wear a uniform, carry a clipboard, act confident. "Hi, I'm here to fix the server room air conditioning." Gains physical access past receptionist. Defense: verify ID against vendor list, escort all visitors, challenge anyone without visible badge.

Dumpster diving →Physical trash for intel — cross-cut shredding + secure disposal policy defends

HTML Smuggling

🚢
HTML Smuggling — Bypassing Email Gateways
Attacker embeds encoded payload in an HTML file as Base64 or JavaScript Blob. When victim opens the HTML in browser, JavaScript decodes and assembles the malicious file client-side — AFTER the email gateway has already delivered it. Email gateway: scanned the HTML file, found no executable — OK. Browser: reconstructs malicious .exe or .zip from encoded data. Defense: block HTML email attachments, disable JavaScript from email HTML, email sandboxing that fully renders HTML.

VCEVoice Call VectorsVishing · Spoofing · AI Voice Cloning

Voice calls exploit the trust people place in telephone communications — particularly when caller ID appears to show a known number. The rise of AI voice cloning has dramatically increased the effectiveness and danger of voice-based attacks.

Voice Attack Types

📞
Vishing — Voice Phishing
Social engineering via phone call. Classic: "IT Support calling — we detected a virus on your computer — I need your credentials to fix it remotely." Or: "Bank fraud department — suspicious transactions — verify your account number and PIN." Caller ID spoofing makes call appear to come from a legitimate number. Highly effective — people trust voices more than email. Defense: never provide credentials over unsolicited call — call back to official number.
🤖
AI Voice Cloning — Deepfake Audio
AI tools can clone a person's voice from as little as 30 seconds of audio sample. Attackers use publicly available audio (YouTube videos, podcasts, earnings calls) to clone executive voices. Scenario: cloned CEO voice calls CFO — "I need you to urgently wire $500K for a confidential acquisition — don't discuss with anyone." British energy company lost £200K in 2019 to AI voice clone. Defense: verbal authentication code (secret word unknown to AI), callback policy, dual-approval regardless of voice instruction.
🔢
Caller ID Spoofing
Caller ID can show any number the attacker wants — not the real originating number. STIR/SHAKEN: emerging standard to verify caller ID authenticity in VoIP calls. Defense: can't trust caller ID alone — verify via callback to known number or out-of-band method. Spoofed government/police numbers especially effective — people don't expect "the IRS" to be fake.
📱
SIM Swapping
Social engineer mobile carrier into transferring victim's phone number to attacker's SIM. Attack: call carrier, pretend to be victim, claim lost phone → "transfer my number to new SIM." Once attacker has victim's number: receive all SMS — including SMS MFA codes. Used to bypass MFA and account recovery. High-profile victims: Twitter CEO Jack Dorsey, multiple crypto executives. Defense: use authenticator app (not SMS) for MFA — set carrier PIN — carrier-level SIM lock.

SUPSupply Chain VectorsVendors · Software · Hardware · MSP

Supply chain attacks compromise a target organization by attacking a trusted vendor, supplier, or component in the chain that delivers products or services to that organization. One compromised vendor can affect thousands of downstream customers simultaneously.

Why Supply Chain Is So Dangerous

# The trust exploitation: Organization: "We trust our vendors — they're vetted." "Software update from SolarWinds? Install it." "Kaseya manages our endpoints — it's secure." Attacker: compromise the vendor → ride the trust relationship 18,000 organizations install → 18,000 backdoors installed SolarWinds 2020: Attacker: compromised SolarWinds build pipeline Malicious code: inserted into legitimate Orion updates Signed with: valid SolarWinds certificate → trusted Deployed to: 18,000+ organizations (US govt, Fortune 500) Dwell time: 9 months before detection Kaseya VSA 2021: Kaseya: RMM tool used by MSPs to manage customer endpoints Attacker: exploited Kaseya zero-day → pushed ransomware to all MSP customers Affected: 1,500+ businesses in 17 countries Ransom: $70M demanded 3CX Supply Chain (2023): 3CX desktop app: communications software Compromised: malicious code in 3CX's own app Signed: with 3CX's own valid code signing certificate Vector: itself a supply chain attack (compromised via another vendor)

Supply Chain Attack Types

Software Supply Chain

Compromise software before it reaches you. Build pipeline attack: malicious code inserted during compilation. Dependency confusion: publish malicious package with same name as internal package — package manager installs attacker's version. Typosquatting in npm/PyPI: "reqeusts" vs "requests" — attacker publishes before victim can typo-check.

Hardware Supply Chain

Hardware modified before delivery. Counterfeit hardware: fake chips with backdoors. Firmware pre-installed: malicious BIOS/firmware on new devices. Modified in transit: interdiction (intelligence agencies). Defense: trusted suppliers, tamper-evident packaging, hardware attestation, firmware verification.

Managed Service Provider (MSP)

MSPs have admin access to many client environments. Compromise one MSP → access to all their clients. Kaseya model: attack MSP's management tool → affect all clients. MSPs are Tier 1 critical vendors — must be assessed stringently. Defense: least privilege for MSP access, monitor MSP access logs, incident response plan if MSP is compromised.

SWCommon Vulnerability TypesSQLi · BOF · RCE · IDOR

SQL Injection (SQLi)

Attacker inserts SQL commands into input fields. Application builds SQL query with attacker's input → executes malicious SQL. Can: read all database data, modify/delete records, bypass authentication. Classic: username = "' OR '1'='1" → login bypass. Defense: parameterized queries (prepared statements) — input validation — WAF.

Buffer Overflow

Write more data into a buffer than it can hold → overwrite adjacent memory → control execution flow. Classic vulnerability in C/C++ applications. Can achieve: RCE (Remote Code Execution). Defense: memory-safe languages (Rust, Go), compiler protections (ASLR, DEP/NX, stack canaries), code analysis.

Cross-Site Scripting (XSS)

Inject malicious JavaScript into web pages viewed by other users. Stored XSS: script saved in database → executes for all viewers. Reflected XSS: script in URL → victim clicks malicious link. DOM-based XSS: modifies page DOM. Can: steal session cookies, redirect to phishing, keylog. Defense: output encoding, Content Security Policy (CSP), input validation.

IDOR — Insecure Direct Object Reference

Access control flaw — user can access another user's objects by guessing/modifying an ID. Example: /account?id=1234 → change to 1235 → see another user's account. Defense: server-side authorization check on every request — not just at login. "Does this user own object 1235?" check.

Race Condition (TOCTOU)

Time-of-Check to Time-of-Use — gap between checking a condition and using a resource is exploited. Check: "does user have permission?" → (attacker changes state) → Use: "use the resource." Classic in file access and privilege operations. Defense: atomic operations, locking, careful concurrency design.

Privilege Escalation

Gain higher privileges than authorized. Vertical: user → admin/root. Horizontal: access another user's account at same level. Methods: exploit kernel/OS vulnerabilities, misconfigured SUID binaries, weak file permissions, token manipulation. Defense: least privilege, patch OS/kernel, regular privilege audits.

SWZero-Day VulnerabilitiesUnknown · No Patch · Critical

A zero-day is a vulnerability that is unknown to the vendor and has no available patch. Name comes from "zero days of patch availability." These are the most valuable and dangerous vulnerabilities — they work against fully patched systems.

# Zero-day timeline: Day -N: Researcher or attacker discovers vulnerability in software Vendor: unaware — no patch exists "Zero-Day" window: vulnerability is unknown to defender Any attack during this window: undefendable via patching Day 0: Disclosure (responsible or not) Day 1+: Vendor develops patch (may take days to months) Day X: Patch released → zero-day becomes "N-day" vulnerability # Zero-day markets: Nation-states: develop or purchase zero-days (millions of $) Bug bounties: responsible disclosure — vendor pays researcher Dark web: criminal markets for 0-days Exploit brokers: Zerodium pays up to $2.5M for iOS zero-day # Defending against zero-days: Virtual patching: WAF/IPS rule blocks exploit pattern before patch exists Defense in depth: even if exploited — what can attacker reach? Behavioral detection: EDR detects exploit behavior not signature Attack surface reduction: less software = fewer zero-days to exploit Sandboxing: isolate vulnerable apps — limit blast radius
